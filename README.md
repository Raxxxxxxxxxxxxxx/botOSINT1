# botOSINT1 — بوت أخبار محافظة الرقة

بوت Telegram (Python 3.12+, aiogram v3) يجمع الأخبار المتعلقة بمحافظة الرقة السورية من مصادر متعددة (RSS، مواقع ويب، قنوات Telegram اختيارياً) وينشرها تلقائياً في قناة/مجموعة محددة، مع كشف تكرار، فلترة جغرافية، تصنيف، وتلخيص اختياري بالذكاء الاصطناعي.

المشروع بُني وفق منهجية بحث وتصميم موثقة قبل الكتابة:

- **المرحلة 1 (البحث):** https://claude.ai/code/artifact/d511f7aa-f3bf-4019-bf7c-03775332c19a
- **المرحلة 2 (Architecture):** https://claude.ai/code/artifact/cb8c8ca3-378f-4f84-a2fc-37865e1ba4fa

## هيكل المشروع

```
config/     إعدادات مقروءة من متغيرات البيئة (.env)
database/   محرك SQLAlchemy async + إنشاء الجداول + بذر المصادر الأولية
models/     نماذج البيانات: Source (مصدر) و NewsItem (خبر)
scrapers/   Source Adapters: RSS / HTML (selectolax) / Telegram (Telethon، اختياري)
filters/    تطبيع عربي (pyarabic) + كشف تكرار (rapidfuzz) + فلترة جغرافية + تصنيف قاعدي
services/   خط المعالجة (Pipeline) + المجدول (APScheduler) + طابور النشر + التلخيص
telegram/   إعداد بوت aiogram والمعالجات (/start)
utils/      تسجيل (loguru) + سياسات إعادة المحاولة (tenacity) + تجزئة الروابط
logs/       ملفات السجل (يتم تدويرها تلقائياً)
main.py     نقطة الدخول: يربط كل شيء ويشغّل حلقة asyncio واحدة
```

## الإعداد المحلي

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # ثم عدّل BOT_TOKEN و TARGET_CHAT_ID على الأقل
python -m database.seed   # يبذر مصادر الرقة المكتشفة بحثياً (معطّلة افتراضياً)
python main.py
```

## إضافة مصدر إخباري جديد

لا حاجة لتعديل أي كود. أضف سطراً في جدول `sources` (عبر سكربت مشابه لـ `database/seed.py` أو أداة إدارة لاحقة):

| الحقل | الوصف |
|---|---|
| `type` | `rss` أو `html` أو `telegram` |
| `url` | رابط RSS، أو رابط صفحة القائمة (للـ HTML)، أو معرّف القناة (للتيليجرام) |
| `list_selector` | (نوع HTML فقط) CSS selector لعناصر الروابط في صفحة القائمة |
| `poll_interval_seconds` | كل كم ثانية يُفحص المصدر |
| `enabled` | فعّله فقط بعد التأكد من صحة الـ selector |

المصادر المبذورة في `database/seed.py` مُدرجة **معطّلة عمداً** لأن البحث (المرحلة 1) أكد وجود هذه المواقع وتغطيتها لأخبار الرقة، لكنه لم يتحقق تقنياً من بنية HTML أو وجود RSS فعلي لكل موقع — يجب فحص كل رابط يدوياً وتحديد الـ selector المناسب قبل التفعيل.

## متغيرات البيئة

راجع `.env.example` — كل متغير موثّق بتعليق يشرح الغرض منه والقيمة الافتراضية.

## النشر على Railway

- المشروع مبني بحيث يعمل ضمن **خدمة واحدة، عملية واحدة** (aiogram + APScheduler + Telethon اختيارياً، كلها في نفس حلقة asyncio) — لا حاجة لـ Redis أو Celery.
- **مهم:** اربط Volume دائم بمسار `./data` (حيث يقع ملف SQLite وجلسة Telethon)، وإلا تُفقد البيانات عند كل إعادة نشر.
- `Dockerfile` متعدد المراحل (`python:3.12-slim`) لتقليل استهلاك الذاكرة، بما يتوافق مع سقف ~512MB على حسابات Railway التجريبية/المجانية.
- ⚠️ ملاحظة اطّلع عليها بحث المرحلة 1: Railway لم يعد لديها خطة مجانية دائمة في 2026 (رصيد تجريبي 5$ فقط) — راجع تقرير المرحلة 1 للتفاصيل قبل الاعتماد الكامل على هذه الاستضافة.

## القيود المعمارية المتعمدة (لا تُضِف بدون مناقشة)

- **لا** متصفح headless (Selenium/Playwright) — يتجاوز حدود الذاكرة.
- **لا** نموذج AI/ML محلي — التلخيص يستدعي API خارجياً فقط (`AI_SUMMARY_ENABLED`)، مع fallback قاعدي إلزامي.
- **لا** Facebook/Instagram/X كمصادر بعد — مؤجلة بقرار من المرحلة 1/2 بسبب هشاشتها القانونية/التقنية في 2026؛ `SourceType` يحجز قيماً لها لإضافتها لاحقاً كنوع Adapter رابع دون تغيير الهيكل.
